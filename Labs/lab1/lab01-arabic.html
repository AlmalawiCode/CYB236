📖 المقدمة (Introduction)

مرحباً بك في أول مختبر للأمن السيبراني! في هذا المختبر، ستتعلم كيفية إعداد بيئة اختبار الاختراق (Penetration Testing) الاحترافية من خلال تثبيت كالي لينكس (Kali Linux) كـ جهاز افتراضي (Virtual Machine) على برنامج Oracle VirtualBox. هذه مهارة أساسية يجب على كل محترف في مجال الأمن السيبراني إتقانها.

💡

لماذا نستخدم الأجهزة الافتراضية؟ (Why Virtual Machines?)

تتيح لك الأجهزة الافتراضية (Virtual Machines - VMs) تشغيل أنظمة تشغيل متعددة على جهاز كمبيوتر واحد بأمان. هذا ضروري لـ:

  • اختبار أدوات الأمان دون التأثير على نظامك الرئيسي
  • إنشاء بيئات مختبر معزولة (Isolated Lab Environments)
  • تعلم أنظمة تشغيل مختلفة
  • أخذ لقطات (Snapshots) لاستعادة الحالات السابقة

🎯 أهداف التعلم (Learning Objectives)

بنهاية هذا المختبر، ستكون قادراً على:

  • فهم ما هو كالي لينكس (Kali Linux) والغرض منه في الأمن السيبراني (Cybersecurity)
  • تثبيت (Install) برنامج Oracle VirtualBox على نظام Windows أو macOS
  • تحميل (Download) صورة الجهاز الافتراضي (VM Image) الرسمية لكالي لينكس
  • استيراد (Import) وتكوين (Configure) كالي لينكس كجهاز افتراضي في VirtualBox
  • تشغيل (Boot) وتسجيل الدخول (Login) بنجاح إلى جهاز كالي لينكس الافتراضي
  • التنقل في واجهة (Interface) كالي لينكس الأساسية

📦 ما هو VirtualBox؟ (What is VirtualBox?)

Oracle VirtualBox هو برنامج افتراضية (Virtualization Software) مجاني ومفتوح المصدر يتيح لك تشغيل كالي لينكس كـ جهاز افتراضي (Virtual Machine) على جهاز الكمبيوتر الخاص بك دون تعديل نظام التشغيل المضيف (Host Operating System).

فوائد استخدام VirtualBox (Benefits of Using VirtualBox)

الفائدة (Benefit) الوصف (Description)
✅ مجاني (Free) برنامج مجاني ومفتوح المصدر بالكامل (Open Source)
✅ متعدد المنصات (Cross-platform) يعمل على Windows و macOS و Linux
✅ اللقطات (Snapshots) حفظ واستعادة حالات الجهاز الافتراضي في أي وقت
✅ العزل (Isolation) الأجهزة الافتراضية معزولة تماماً عن نظامك المضيف
✅ سهل الاستخدام (Easy to Use) واجهة رسومية سهلة الاستخدام (GUI - Graphical User Interface)
🌐 زيارة الموقع الرسمي لـ VirtualBox

💻 متطلبات النظام (System Requirements)

قبل البدء، تأكد من أن جهاز الكمبيوتر الخاص بك يفي بهذه المتطلبات:

المكون (Component) الحد الأدنى (Minimum) الموصى به (Recommended)
المعالج (CPU) معالج 64-bit مع دعم الافتراضية
(Virtualization Support)		 Intel i5/AMD Ryzen 5 أو أفضل
الذاكرة العشوائية (RAM) 8 جيجابايت (GB) 16 جيجابايت أو أكثر
التخزين (Storage) 50 جيجابايت مساحة فارغة 100+ جيجابايت SSD
نظام التشغيل (Operating System) Windows 10/11 أو macOS 10.15+ أحدث إصدار (Latest Version)
🔧

هام: تفعيل الافتراضية (Enable Virtualization)

يجب تفعيل الافتراضية (Virtualization) في إعدادات BIOS/UEFI لجهاز الكمبيوتر. للتحقق على Windows:

  1. افتح مدير المهام (Task Manager) بالضغط على (Ctrl + Shift + Esc)
  2. انتقل إلى علامة التبويب الأداء (Performance)
  3. انقر على وحدة المعالجة المركزية (CPU)
  4. ابحث عن "Virtualization: Enabled"

🛠️ الجزء الأول: تثبيت VirtualBox (Part 1: Installing VirtualBox)

أولاً، نحتاج إلى تثبيت (Install) برنامج VirtualBox على جهاز الكمبيوتر الخاص بك. اختر نظام التشغيل الخاص بك أدناه:

🪟 Windows (ويندوز)
🍎 macOS (ماك)

🪟 التثبيت على Windows (Windows Installation)

الخطوة 1: تحميل VirtualBox (Step 1: Download VirtualBox)

  1. افتح متصفح الويب (Web Browser) الخاص بك (Chrome أو Firefox أو Edge)
  2. انتقل إلى صفحة تحميل (Download Page) VirtualBox الرسمية:

    ⬇️ صفحة تحميل VirtualBox (VirtualBox Downloads Page)
  3. انقر على "Windows hosts" لتحميل المثبت (Installer)
  4. احفظ الملف في مجلد التنزيلات (Downloads folder)

الخطوة 2: تشغيل المثبت (Step 2: Run the Installer)

  1. افتح مجلد التنزيلات (Downloads)
  2. ابحث عن الملف المسمى VirtualBox-x.x.x-xxxxxx-Win.exe
  3. انقر نقراً مزدوجاً (Double-click) على المثبت لتشغيله
  4. إذا ظهرت رسالة التحكم في حساب المستخدم (UAC - User Account Control)، انقر نعم (Yes)

الخطوة 3: معالج التثبيت (Step 3: Installation Wizard)

  1. شاشة الترحيب (Welcome Screen): انقر Next
  2. الإعداد المخصص (Custom Setup): احتفظ بالإعدادات الافتراضية (Default Settings)، انقر Next
  3. تحذير الشبكة (Network Warning): سيظهر تحذير حول قطع اتصال الشبكة مؤقتاً - هذا طبيعي! انقر Yes
  4. التبعيات المفقودة (Missing Dependencies): إذا طُلب منك، انقر Yes لتثبيت Python Core
  5. جاهز للتثبيت (Ready to Install): انقر Install
  6. انتظر حتى يكتمل التثبيت
  7. انقر Finish لإغلاق المثبت

الخطوة 4: التحقق من التثبيت (Step 4: Verify Installation)

  1. انقر على قائمة ابدأ (Start Menu)
  2. اكتب VirtualBox
  3. انقر على Oracle VM VirtualBox
  4. يجب أن تفتح نافذة VirtualBox Manager
✅

تم بنجاح! (Success!)

تم تثبيت VirtualBox الآن على جهاز Windows الخاص بك!

🍎 التثبيت على macOS (macOS Installation)

الخطوة 1: تحميل VirtualBox (Step 1: Download VirtualBox)

  1. افتح متصفح الويب (Web Browser) الخاص بك (Safari أو Chrome أو Firefox)
  2. انتقل إلى صفحة تحميل VirtualBox الرسمية:

    ⬇️ صفحة تحميل VirtualBox (VirtualBox Downloads Page)
  3. انقر على "macOS / Intel hosts" للتحميل
    (لأجهزة M1/M2/M3، انظر الملاحظة أدناه)
📝

ملاحظة لمستخدمي Apple Silicon (M1/M2/M3)

إذا كان لديك جهاز Mac بمعالج Apple Silicon، قد تحتاج إلى استخدام برنامج افتراضية بديل مثل UTM أو Parallels. راجع وثائق كالي (Kali Documentation) لدعم ARM.

الخطوة 2: تثبيت VirtualBox (Step 2: Install VirtualBox)

  1. افتح Finder وانتقل إلى مجلد التنزيلات (Downloads)
  2. ابحث عن الملف المسمى VirtualBox-x.x.x-xxxxxx-macOS.dmg
  3. انقر نقراً مزدوجاً على ملف .dmg لتحميله (Mount)
  4. انقر نقراً مزدوجاً على VirtualBox.pkg لبدء المثبت
  5. انقر Continue على شاشة الترحيب
  6. انقر Install
  7. أدخل كلمة مرور Mac (Mac Password) عند المطالبة
  8. انقر Install Software

الخطوة 3: السماح بامتداد النظام (Step 3: Allow System Extension) - هام!

⚠️

إعدادات أمان macOS (macOS Security Settings)

يتطلب macOS السماح بـ امتداد النظام (System Extension) الخاص بـ VirtualBox:

  1. قد تظهر نافذة منبثقة تقول إن امتداد النظام تم حظره (blocked)
  2. انتقل إلى System Preferences (تفضيلات النظام) ← Security & Privacy (الأمان والخصوصية)
  3. انقر على علامة التبويب General (عام)
  4. انقر Allow (السماح) بجانب الرسالة حول Oracle
  5. قد تحتاج إلى إعادة تشغيل Mac (Restart)

الخطوة 4: التحقق من التثبيت (Step 4: Verify Installation)

  1. افتح Finder
  2. انتقل إلى Applications (التطبيقات)
  3. انقر نقراً مزدوجاً على VirtualBox
  4. يجب أن تفتح نافذة VirtualBox Manager
✅

تم بنجاح! (Success!)

تم تثبيت VirtualBox الآن على جهاز Mac الخاص بك!

📥 الجزء الثاني: تحميل كالي لينكس (Part 2: Downloading Kali Linux)

الآن بعد تثبيت VirtualBox، دعنا نحمل صورة الجهاز الافتراضي (Virtual Machine Image) لكالي لينكس.

الخطوة 1: الذهاب إلى صفحة تحميل كالي (Step 1: Go to Kali Download Page)

  1. افتح متصفح الويب الخاص بك
  2. انتقل إلى صفحة تحميل كالي لينكس الرسمية:

    🐉 تحميل كالي لينكس - الأجهزة الافتراضية (Kali Linux Downloads - Virtual Machines)

الخطوة 2: اختيار صورة VirtualBox (Step 2: Select VirtualBox Image)

  1. في صفحة التحميل، ابحث عن قسم "Virtual Machines" (الأجهزة الافتراضية)
  2. ابحث عن خيار VirtualBox
  3. انقر على زر Download (تحميل) للإصدار 64-bit
معلومات الملف (File Information) التفاصيل (Details)
الصيغة (Format) .7z (أرشيف مضغوط 7-Zip)
يحتوي على (Contains) ملف .vbox + قرص افتراضي .vmdk
الحجم (Size) ~3-4 جيجابايت (مضغوط)
المعمارية (Architecture) 64-bit (amd64)
⏳

وقت التحميل (Download Time)

قد يستغرق التحميل 10-30 دقيقة حسب سرعة الإنترنت لديك. يرجى الصبر!

الخطوة 3: التحقق من التحميل - موصى به (Step 3: Verify Download - Recommended)

🔒

أفضل ممارسات الأمان (Security Best Practice)

تحقق دائماً من الملفات المحملة للتأكد من عدم التلاعب بها. قارن رمز SHA256 Checksum المعروض في صفحة التحميل مع ملفك:

Windows (PowerShell):

Get-FileHash .\kali-linux-*-virtualbox-amd64.7z -Algorithm SHA256

macOS (Terminal):

shasum -a 256 ~/Downloads/kali-linux-*-virtualbox-amd64.7z

الخطوة 4: فك ضغط الأرشيف (Step 4: Extract the Archive)

الملف المحمّل هو أرشيف مضغوط بصيغة .7z. يجب فك ضغطه قبل إضافته إلى VirtualBox.

  1. إذا لم يكن لديك برنامج 7-Zip:
    • Windows: حمّله من 7-zip.org وثبّته
    • macOS: ثبّت Keka أو استخدم: brew install p7zip
  2. انقر بزر الماوس الأيمن على ملف .7z المحمّل
  3. اختر 7-Zip → Extract Here (فك الضغط هنا)
  4. انتظر حتى يكتمل فك الضغط — سيُنشئ مجلدًا يحتوي على ملفات الجهاز الافتراضي
📂

بعد فك الضغط (After Extraction)

ستجد مجلدًا يحتوي على ملف .vbox وملف .vmdk. ملف .vbox هو إعدادات VirtualBox، وملف .vmdk هو القرص الصلب الافتراضي.

📦 الجزء الثالث: إضافة كالي لينكس إلى VirtualBox (Part 3: Adding Kali Linux)

الآن سنقوم بإضافة صورة كالي لينكس المستخرجة كـ جهاز افتراضي (Virtual Machine) في VirtualBox.

الخطوة 1: فتح VirtualBox (Step 1: Open VirtualBox)

  1. قم بتشغيل Oracle VM VirtualBox من:
    • Windows: قائمة ابدأ (Start Menu) → VirtualBox
    • macOS: التطبيقات (Applications) → VirtualBox

الخطوة 2: إضافة جهاز كالي لينكس (Step 2: Add Kali Linux VM)

  1. في VirtualBox، انتقل إلى Machine (جهاز) → Add (إضافة)
    أو استخدم اختصار لوحة المفاتيح: Ctrl + A (Windows) / Cmd + A (macOS)
  2. انتقل إلى المجلد المستخرج من الخطوة السابقة
  3. اختر ملف .vbox (مثال: kali-linux-*-virtualbox-amd64.vbox)
  4. انقر Open (فتح)

الخطوة 3: مراجعة إعدادات الجهاز الافتراضي (Step 3: Review VM Settings)

ستظهر إعدادات (Settings) الجهاز الافتراضي. راجعها وعدّلها اختيارياً:

الإعداد (Setting) الافتراضي (Default) الموصى به (Recommended)
الاسم (Name) Kali Linux احتفظ به أو خصصه (Keep or customize)
المعالج (CPU) 2 نواة (cores) زيادة إلى 4 إن أمكن
الذاكرة (RAM) 2048 ميجابايت (MB) زيادة إلى 4096 ميجابايت إن أمكن
الشبكة (Network) NAT احتفظ بـ NAT
💡

نصيحة للأداء (Performance Tip)

إذا كان جهاز الكمبيوتر الخاص بك يحتوي على 16 جيجابايت أو أكثر من الذاكرة العشوائية (RAM)، خصص 4 جيجابايت (4096 MB) لكالي للحصول على أداء أفضل.

الخطوة 4: التحقق من إضافة الجهاز (Step 4: Verify the VM is Added)

  1. يجب أن يظهر "Kali Linux" الآن في VirtualBox Manager
  2. الجهاز الافتراضي جاهز — لا حاجة لانتظار الاستيراد لأن الملفات مستخرجة مسبقًا
✅

تم بنجاح! (Success!)

تمت إضافة الجهاز الافتراضي لكالي لينكس بنجاح!

🚀 الجزء الرابع: التشغيل الأول وتسجيل الدخول (Part 4: First Boot and Login)

الخطوة 1: تشغيل الجهاز الافتراضي (Step 1: Start the Virtual Machine)

  1. في VirtualBox Manager، اختر Kali Linux
  2. انقر على زر Start (السهم الأخضر) أو انقر نقراً مزدوجاً على Kali Linux

الخطوة 2: انتظار التشغيل (Step 2: Wait for Boot)

  1. ستفتح نافذة جديدة تعرض عملية التشغيل (Boot Process) لكالي لينكس
  2. انتظر حتى يكتمل تشغيل النظام (سترى شاشة تسجيل الدخول Login Screen)

الخطوة 3: تسجيل الدخول (Step 3: Log In)

استخدم بيانات الاعتماد الافتراضية (Default Credentials):

🔑 بيانات تسجيل الدخول الافتراضية (Default Login Credentials)

اسم المستخدم (Username) kali
كلمة المرور (Password) kali
⚠️

تحذير أمني (Security Warning)

كلمة المرور الافتراضية kali معروفة للجميع. في بيئة الإنتاج (Production Environment)، يجب عليك تغيير كلمة المرور فوراً!

الخطوة 4: استكشاف سطح المكتب (Step 4: Explore the Desktop)

بعد تسجيل الدخول، سترى سطح مكتب (Desktop) كالي لينكس مع:

  • قائمة التطبيقات (Applications Menu) - الزاوية العلوية اليسرى - للوصول إلى جميع أدوات الأمان
  • شريط النظام (System Tray) - الزاوية العلوية اليمنى - الشبكة والصوت والطاقة
  • مدير الملفات (File Manager) - تصفح الملفات
  • الطرفية (Terminal) - واجهة سطر الأوامر (Command Line Interface)

الخطوة 5: تغيير كلمة المرور - موصى به (Step 5: Change Password - Recommended)

  1. افتح Terminal (الطرفية) - انقر على أيقونة الطرفية أو انقر بالزر الأيمن على سطح المكتب ← Open Terminal Here
  2. اكتب الأمر التالي واضغط Enter: 
    passwd
  3. أدخل كلمة المرور الحالية: kali
  4. أدخل كلمة المرور الجديدة (مرتين)

✅ مهام التحقق (Verification Tasks)

أكمل هذه المهام للتحقق من أن جهاز كالي لينكس الافتراضي (Kali Linux Virtual Machine) يعمل بشكل صحيح:

المهمة 1: فحص الشبكة (Task 1: Check Network)

افتح Terminal وشغّل:

ping -c 4 google.com

المتوقع (Expected): يجب أن ترى ردود ping replies مع أوقات الاستجابة.

المهمة 2: تحديث النظام (Task 2: Update System)

افتح Terminal وشغّل:

sudo apt update

🔧 استكشاف الأخطاء وإصلاحها (Troubleshooting)

إذا واجهت مشاكل، تحقق من هذه المشاكل الشائعة وحلولها:

❌ خطأ "VT-x is not available" (VT-x غير متاح)

المشكلة (Problem): الافتراضية (Virtualization) معطلة في BIOS/UEFI.

الحل (Solution):

  1. أعد تشغيل (Restart) جهاز الكمبيوتر
  2. ادخل إعداد BIOS/UEFI (اضغط F2 أو F10 أو F12 أو Del أثناء التشغيل)
  3. ابحث عن إعدادات الافتراضية (VT-x, AMD-V, SVM, أو Virtualization Technology)
  4. فعّل (Enable) الافتراضية
  5. احفظ (Save) واخرج من BIOS
❌ شاشة سوداء بعد التشغيل (Black Screen After Boot)

المشكلة (Problem): مشكلة في إعدادات العرض (Display Settings).

الحل (Solution):

  1. أوقف تشغيل (Power Off) الجهاز الافتراضي
  2. انتقل إلى Settings ← Display
  3. زِد Video Memory إلى 128 ميجابايت
  4. عطّل (Disable) 3D Acceleration
  5. حاول التشغيل مرة أخرى
❌ أداء بطيء (Slow Performance)

المشكلة (Problem): موارد (Resources) غير كافية مخصصة للجهاز الافتراضي.

الحل (Solution):

  1. أوقف تشغيل الجهاز الافتراضي
  2. انتقل إلى Settings ← System
  3. زِد RAM (الذاكرة) إلى 4096 ميجابايت
  4. انتقل إلى Settings ← Processor
  5. زِد أنوية CPU (المعالج) إلى 2-4
❌ الشبكة لا تعمل (Network Not Working)

المشكلة (Problem): مشكلة في تكوين محول الشبكة (Network Adapter).

الحل (Solution):

  1. أوقف تشغيل الجهاز الافتراضي
  2. انتقل إلى Settings ← Network
  3. تأكد من تفعيل (Enabled) Adapter 1
  4. اضبط "Attached to" على NAT
  5. انقر OK وشغّل الجهاز الافتراضي
❌ فشل الاستيراد (Import Failed)

المشكلة (Problem): تحميل تالف (Corrupted Download) أو فشل فك الضغط أو مساحة قرص غير كافية.

الحل (Solution):

  1. احذف (Delete) ملف .7z المحمل والمجلد المستخرج
  2. أعد التحميل من موقع كالي الرسمي (Official Kali Website)
  3. تحقق من رمز SHA256 Checksum
  4. تأكد من وجود 50 جيجابايت على الأقل من المساحة الفارغة (Free Space)
  5. تأكد من تثبيت برنامج 7-Zip بشكل صحيح لفك الضغط
