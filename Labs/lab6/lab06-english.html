Lab 6: Authentication & Session Management Attacks

Topic 6 — Web Application Security Fundamentals II

DVWA + Burp Suite + Hydra (Kali)

Lab Purpose

This lab demonstrates how authentication and session management weaknesses can be exploited. Students will perform a brute force attack on a login form, analyze weak session tokens, and observe the consequences of poor session handling — directly connecting to the theory of Authentication, Access Control, and Session Management from Topic 4.

Tools Required

DVWA Burp Suite (Intruder) Hydra (Kali)

Planned Tasks

  • Task 1 — Brute Force Login: Use Burp Suite Intruder or Hydra to brute force the DVWA login page. Observe how a weak password policy allows an attacker to guess credentials automatically.
  • Task 2 — Weak Session IDs: Use the DVWA "Weak Session IDs" module. Generate multiple session tokens and analyze their pattern. Demonstrate how a predictable session ID can be hijacked.
  • Task 3 — Session Fixation Observation: Observe how session tokens behave before and after login. Discuss the concept of session fixation and why session regeneration on login is important.

Learning Outcomes

  • Understand how brute force attacks exploit weak authentication
  • Recognize the risk of predictable or weak session tokens
  • Connect these attacks to the defensive principles covered in Topic 4 (Input Validation, Defense-in-Depth)
Instructor Note: Set DVWA security level to LOW for Tasks 1 and 2. For Task 3, compare LOW vs MEDIUM behavior to show how a simple fix (randomizing session IDs) mitigates the risk.
