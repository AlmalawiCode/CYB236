Lab 7: Automated Scanning with Nikto & OWASP ZAP

Topic 7 — Secure Development Methodologies I

Nikto (Kali) + OWASP ZAP (Kali)

Lab Purpose

This lab introduces automated security testing tools — a core part of the Security Testing methodology covered in Topic 7. Students will use Nikto and OWASP ZAP to scan DVWA automatically, compare results with their manual findings from previous labs, and understand how security professionals use automated tools as part of a broader testing strategy.

Tools Required

Nikto (pre-installed in Kali) OWASP ZAP (pre-installed in Kali) DVWA

Planned Tasks

  • Task 1 — Nikto Scan: Run Nikto against the DVWA target. Review the output — which vulnerabilities did Nikto detect automatically? Which ones did it miss?
  • Task 2 — OWASP ZAP Active Scan: Configure OWASP ZAP to proxy through the browser, spider DVWA, then run an active scan. Review the alerts generated.
  • Task 3 — Compare Results: Compare Nikto vs ZAP findings. Identify which tool found what. Discuss the limitations of automated scanning vs manual testing.
  • Task 4 — Document Findings: Write a brief vulnerability summary table listing each finding, severity, and tool that detected it.

Learning Outcomes

  • Use Nikto and OWASP ZAP for automated web application scanning
  • Understand the strengths and limitations of automated tools
  • Connect automated scanning to the Security Testing methodology from Topic 7
  • Practice documenting findings in a structured format
Instructor Note: Both Nikto and OWASP ZAP are pre-installed in Kali Linux. Target DVWA running locally. Make sure DVWA is set to LOW security before scanning so students see maximum findings.
