Part C
Intruder & Password Attacks (Lab 7)
45 pts

💡 Expected Answers — Burp Intruder against DVWA (Reference)

  • Q8 — Attack mode: Sniper — correct because only one payload position (the password field) is being tested with a single wordlist
  • Q9 — Identifying correct password: The correct entry has a noticeably different (longer) response length compared to all failed attempts; the password is password
  • Q10 — Security levels: Medium adds a time delay (sleep) between responses, slowing the attack; High adds a CSRF token (user_token) that must be extracted and resubmitted for each request, effectively breaking a standard Intruder attack
  • Q11 — Comparison: Dictionary uses a pre-built wordlist (faster, limited coverage); Brute force tries every possible combination (guaranteed coverage, extremely slow). Lab 7 used dictionary because a wordlist was provided and the target password was weak/common
  • Q12 — Defenses: Account lockout, rate limiting, and CAPTCHA — each makes automated credential testing impractical

📝 Grading Notes for Instructors

  • For Q1, accept localhost:8080 as equivalent to 127.0.0.1:8080.
  • For Q2, accept any extension that routes browser traffic to Burp (FoxyProxy Standard, FoxyProxy Basic, or manual Firefox proxy configuration); award the advantage point if a genuine practical benefit is stated.
  • For Q6, the DVWA headers a student finds may vary; accept any standard security header that is legitimately missing. Do not penalize for listing a header that was present if the student's explanation of the attack it prevents is correct.
  • For Q9, the password password is hardcoded in DVWA; award full marks for any student who correctly reports the response-length method, even if they type the password slightly differently (e.g., lowercase vs. mixed case).
  • For Q10, if a student reports that High security prevented the attack entirely without explaining the CSRF token mechanism, award partial credit (4 pts) for correctly observing the outcome.
  • For Q12, accept multi-factor authentication (MFA) as a valid alternative to CAPTCHA; award the explanation point if the student correctly states how MFA defeats credential-only attacks.
  • Screenshots need not be high-resolution; readability is the key criterion.
