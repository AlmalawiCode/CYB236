💡 What is Burp Suite?

Burp Suite is a web application security testing platform developed by PortSwigger. It is the industry-standard tool used by penetration testers, bug bounty hunters, and security researchers to find vulnerabilities in web applications.

At its core, Burp Suite works as an intercepting proxy: it sits between your browser and any website, giving you complete visibility and control over every HTTP request and response that passes through.

Burp Suite is Pre-installed in Kali Linux

You do not need to install Burp Suite. It ships with Kali Linux. Launch it from the terminal with burpsuite or from the application menu: Applications → Web Application Analysis → burpsuite.

🔁 How a Proxy Works

To understand Burp Suite, you need to understand what a proxy is and why it gives you power over HTTP traffic.

The Normal Connection (Without a Proxy)

When you type a URL in Firefox and press Enter, your browser sends an HTTP request directly to the web server. The server responds, and the page loads. You see only the rendered page — not the raw request or response.

The Proxy Connection (With Burp Suite)

When you route Firefox through Burp Suite, every request passes through Burp first. Burp receives it, records it, optionally modifies it, then forwards it to the server. The server's response comes back to Burp, which records it and passes it back to Firefox. You see everything in raw detail.

Step-by-Step: What Happens When You Load a Page Through Burp

  1. You type a URL in Firefox and press Enter
  2. FoxyProxy (browser extension) intercepts the outgoing request and redirects it to 127.0.0.1:8080 instead of the website directly
  3. Burp Suite is listening on 127.0.0.1:8080 — it receives the raw HTTP request
  4. Burp records the request in its HTTP History. If Intercept is ON, it pauses here and waits for you to read/modify/forward it
  5. Burp forwards the request to the actual target server
  6. The server sends its HTTP response back to Burp
  7. Burp records the response and passes it back to Firefox
  8. Firefox renders the page as normal — but Burp has a full copy of everything
💡

Key Insight

Burp Suite does not change what Firefox displays — it changes what you can see and control. From Firefox's perspective, the page loads normally. But Burp has a complete record of every raw request and response, including cookies, hidden fields, and parameters that never appear in the browser UI.

🦊 Why FoxyProxy?

Firefox has a built-in proxy setting: Settings → Network Settings → Manual proxy configuration. You could use that. So why install FoxyProxy instead?

FeatureFirefox Built-in ProxyFoxyProxy Extension
Enable/DisableRequires opening Settings → Network Settings each timeOne click on the toolbar icon
Multiple ProfilesOne proxy at a time, manual switchingSave multiple profiles, switch instantly
Visual indicatorNo icon change when proxy is activeIcon changes color when proxy is ON
Pattern matchingBasic exclusions onlyRoute only specific URLs through proxy
After-lab cleanupEasy to forget to remove → no internetEasier to notice and disable
⚠️

Always Disable FoxyProxy After the Lab

When FoxyProxy is active and pointing to Burp Suite, all your Firefox traffic goes through Burp. If you close Burp but leave FoxyProxy on, Firefox will have no internet access. Always click the FoxyProxy icon and select "Disable FoxyProxy" at the end of every lab session.

⚙️ Part 1: Launch Burp Suite

Step 1: Open Burp Suite

Open a terminal in Kali Linux and run:

burpsuite

Or find it in the application menu: Applications → Web Application Analysis → burpsuite

Step 2: Create a Temporary Project

  1. On the welcome screen, select Temporary project
  2. Click Next
  3. Select Use Burp defaults
  4. Click Start Burp
  5. Wait for the Burp dashboard to fully load (this may take 30–60 seconds)
💡

Temporary vs Saved Projects

A Temporary project stores everything in memory and is discarded when you close Burp. For course labs this is fine. Professional testers use saved projects to keep records of engagements across sessions.

⚠️

Burp Starts Slowly

Burp Suite is a Java application and can take 30–90 seconds to fully load, especially in a VM. If it appears frozen, wait. If you get a Java error: run sudo apt install -y default-jre in a terminal, then try again.

🔎 Part 2: Verify the Proxy Listener

Before configuring the browser, confirm that Burp Suite is actively listening for incoming traffic.

  1. In Burp Suite, click the Proxy tab at the top
  2. Click Proxy settings (or Options in older versions)
  3. Under Proxy Listeners, confirm you see an entry for 127.0.0.1:8080 with the status Running
✅

What You Should See

The listener entry should show: Interface: 127.0.0.1:8080  —  Running: checked. This means Burp is ready to receive proxy traffic on that address and port.

💡

What Does 127.0.0.1:8080 Mean?

127.0.0.1 is the loopback address — it always means "this machine." Port 8080 is the port Burp listens on. When you tell FoxyProxy to send traffic to 127.0.0.1:8080, it sends it to Burp Suite running on your own computer. No traffic leaves the machine to reach Burp — it stays local.

🦊 Part 3: Install FoxyProxy

  1. Open Firefox
  2. In the address bar, type: about:addons and press Enter
  3. Click the search icon (or go to the Extensions tab and search)
  4. Search for: FoxyProxy Standard
  5. Click the result from the official listing
  6. Click Add to Firefox
  7. In the permission dialog, click Add
  8. FoxyProxy icon will appear in the Firefox toolbar (a fox icon)
💡

FoxyProxy Standard vs Basic

Install FoxyProxy Standard (not Basic). Standard supports URL pattern matching, which lets you route only specific traffic through Burp while keeping other browser traffic unaffected.

⚙️ Part 4: Configure FoxyProxy

Step 1: Open FoxyProxy Options

  1. Click the FoxyProxy icon in the Firefox toolbar
  2. Click Options (opens a settings tab)

Step 2: Add the Burp Proxy Profile

  1. Click Add to create a new proxy entry
  2. Fill in the fields exactly as follows:
    FieldValueWhy
    TitleBurp ProxyA memorable name for this profile
    Proxy TypeHTTPBurp uses HTTP proxy protocol
    IP Address127.0.0.1Burp is running on this machine (loopback)
    Port8080Burp's default proxy listener port

Step 3: Set the URL Pattern

  1. In the proxy entry, find the URL Patterns section
  2. Add a pattern: *
  3. This wildcard routes all browser traffic through Burp — nothing is excluded

Step 4: Save and Activate

  1. Click Save
  2. Click the FoxyProxy icon in the toolbar
  3. Select Burp Proxy to activate it
  4. The FoxyProxy icon changes — this confirms the proxy is now active
✅

Configuration Summary

FoxyProxy is now configured to send all Firefox traffic to 127.0.0.1:8080 — where Burp Suite is listening. The browser and the proxy are now connected.

🌍 Part 5: Understanding the IP Address Configuration

There are two different IP addresses involved in this setup, and they serve completely different purposes. Confusing them is one of the most common causes of Burp not working.

The Two Addresses

AddressWhat It IsUsed For
127.0.0.1 Loopback (this machine) FoxyProxy's proxy address — tells the browser to send traffic to Burp, which is running locally
Your machine IP
(e.g., 10.0.2.15)		 Your Kali machine's network IP The target URL — the address you type in Firefox to reach a locally running app like DVWA

How to Get Your Machine IP

# Run this in a terminal to find your Kali IP address
hostname -I
10.0.2.15
⚠️

Why Not Use localhost or 127.0.0.1 as the Target URL?

When your browser sends a request to http://127.0.0.1 or http://localhost, some browsers and operating systems treat this as a "local" destination and bypass the proxy entirely. The request goes directly to the local app, never passing through Burp. You will see no traffic in Burp's HTTP history.

Using http://10.0.2.15 (the real IP) forces the request to go through the proxy because it looks like a normal network request.

💡

The Rule to Remember

FoxyProxy destination (where traffic goes to reach Burp) = 127.0.0.1:8080
Browser target URL (the app you want to test) = http://<your-IP> from hostname -I

Visual Summary

Firefox types:  http://10.0.2.15  ← your machine IP (target app)

FoxyProxy routes to:  127.0.0.1:8080  ← Burp listener

Burp forwards to:  http://10.0.2.15  ← the actual target

Result: Burp sits in the middle and sees everything ✅

🎯 Part 6: Intercept Your First Request

Now you will verify the entire setup by intercepting a live HTTP request. This confirms FoxyProxy, Burp Suite, and the IP configuration are all working correctly.

💡

What You Need for This Exercise

You need any locally running HTTP web application. If you have DVWA running (sudo docker start dvwa), use that. Otherwise, any HTTP page served locally will work. The important thing is to use the machine IP (hostname -I) not localhost in the browser.

Step 1: Confirm Burp and FoxyProxy Are Both Active

  • Burp Suite is open and the proxy listener shows 127.0.0.1:8080 as Running
  • FoxyProxy icon shows the Burp Proxy profile is active

Step 2: Get Your Machine IP

hostname -I

Note the IP address shown (e.g., 10.0.2.15).

Step 3: Turn Intercept ON in Burp

  1. In Burp Suite, click the Proxy tab
  2. Click the Intercept sub-tab
  3. If the button reads "Intercept is off", click it once
  4. It should now read "Intercept is on" (highlighted)

Step 4: Trigger a Request from Firefox

  1. In Firefox, type http://<your-ip> in the address bar (e.g., http://10.0.2.15)
  2. Press Enter
  3. Firefox will freeze and not load the page — this is expected and correct
  4. Firefox is waiting because Burp has caught the request mid-flight

Step 5: Inspect the Intercepted Request in Burp

  1. Switch to Burp Suite → Proxy → Intercept
  2. You will see the raw HTTP request in the panel:
GET / HTTP/1.1
Host: 10.0.2.15
User-Agent: Mozilla/5.0 ...
Accept: text/html,application/xhtml+xml...
Connection: keep-alive

This is the raw HTTP request exactly as your browser sent it. You can read every header, modify any value, or block it entirely.

Step 6: Forward the Request

  1. Click Forward in Burp's Intercept panel
  2. Firefox will receive the response and the page will load
  3. Repeat clicking Forward if the page makes additional requests (CSS, JS, images)

Step 7: Verify HTTP History

  1. In Burp, click Proxy → HTTP history sub-tab
  2. You should see rows of captured requests
  3. Click any row to see the full request and response in the panels below

Step 8: Turn Intercept OFF

  1. Click the Intercept button so it reads "Intercept is off"
  2. Now traffic flows freely through Burp without pausing — Burp still records everything in HTTP history, but does not block each request
✅

Setup Complete

If you saw an HTTP request in Burp's Intercept panel and requests appeared in HTTP History, your entire proxy chain is working: Firefox → FoxyProxy → Burp Suite → Target. You are ready to use Burp Suite for security testing in Lab 06.

🔧 Troubleshooting

❌ Burp Suite won't start or shows a Java error

Solution: Run in terminal: sudo apt install -y default-jre, then try burpsuite again. If Burp loads but is very slow, close other applications to free RAM — Burp needs at least 1 GB.

❌ Firefox can't load any pages after enabling FoxyProxy

Solution: FoxyProxy is active but Burp is not running (or not listening). Make sure Burp Suite is open and the proxy listener shows 127.0.0.1:8080 as Running. Also confirm Intercept is OFF — if it is ON, the browser waits for you to click Forward on every single request.

❌ No requests appearing in Burp's HTTP History

Solution:

  • Confirm FoxyProxy is enabled and the Burp Proxy profile is selected (not "Disable FoxyProxy")
  • Confirm you are browsing to http://<your-ip> (from hostname -I), not localhost or 127.0.0.1
  • Confirm Burp's proxy listener is on 127.0.0.1:8080 and shows Running
  • Use http:// not https:// — HTTPS requires installing Burp's CA certificate
❌ Firefox is frozen and nothing happens after typing the URL

Solution: Burp's Intercept is ON. Go to Burp → Proxy → Intercept and click Forward to release the request (or turn Intercept OFF). The browser freezes whenever Intercept is ON and Burp is holding a request waiting for you to act on it.

❌ After the lab, Firefox has no internet access

Solution: FoxyProxy is still routing traffic to Burp, which is no longer running. Click the FoxyProxy icon in the Firefox toolbar and select "Disable FoxyProxy". This must be done at the end of every lab session.

❌ Intercepted request shows wrong host or localhost instead of IP

Solution: Make sure you typed http://<your-ip> in the Firefox address bar (not localhost). The Host header in the intercepted request should show your IP address (e.g., Host: 10.0.2.15), not localhost.
