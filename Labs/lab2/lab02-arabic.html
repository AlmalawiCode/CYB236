المقدمة

في هذا المختبر ستخطو خطواتك الأولى في استطلاع تطبيقات الويب (Web Application Reconnaissance) — وهي مرحلة جمع المعلومات التي تحدث قبل أي اختبار أمني. ستستكشف تطبيق ويب مستهدف باستخدام متصفح الويب فقط وأدوات المطور (Developer Tools) المدمجة فيه.

التطبيق المستهدف هو OWASP Juice Shop، وهو متجر عصائر إلكتروني مصمم عمداً ليكون معرضاً للثغرات، مبني بتقنيات حديثة درستها في الفصل 3.

💡

ما هو OWASP Juice Shop؟

Juice Shop هو تطبيق ويب حديث مصمم خصيصاً لتعليم الأمن السيبراني. يستخدم:

  • Angular — إطار عمل تطبيقات الصفحة الواحدة (SPA)
  • Node.js / Express — خادم ويب بلغة JavaScript
  • REST API بصيغة بيانات JSON
  • قاعدة بيانات SQLite
⚠️

تنبيه قانوني مهم

يجب إجراء الاستطلاع فقط على التطبيقات التي تملكها أو لديك إذن كتابي لاختبارها. في هذا المختبر، يعمل Juice Shop محلياً على جهازك، لذا أنت مخوّل بالكامل.

أهداف التعلم

بنهاية هذا المختبر، ستتمكن من:

  • إعداد وتشغيل OWASP Juice Shop باستخدام Docker على Kali Linux
  • تحديد التقنيات المستخدمة في تطبيق ويب باستخدام أدوات المتصفح
  • استخدام أدوات المطور (Developer Tools) لمراقبة طلبات REST API
  • قراءة وفهم بيانات JSON المتبادلة بين العميل والخادم
  • فحص التخزين في المتصفح (الكوكيز، localStorage، sessionStorage)

الأدوات المستخدمة في هذا المختبر

الأداة الاستخدام
Docker تشغيل تطبيق Juice Shop محلياً
Firefox + أدوات المطور (F12) فحص طلبات الشبكة وبيانات JSON والتخزين

الجزء 1: إعداد OWASP Juice Shop

سنقوم بتشغيل Juice Shop باستخدام Docker داخل جهازك الافتراضي Kali.

الخطوة 1: تثبيت Docker

افتح الطرفية (Terminal) في Kali VM ونفّذ هذه الأوامر واحداً تلو الآخر:

# تحديث الحزم وتثبيت Docker
sudo apt update
sudo apt install -y docker.io

# تشغيل Docker
sudo systemctl start docker
sudo systemctl enable docker

الخطوة 2: تحميل وتشغيل Juice Shop

# تحميل Juice Shop (قد يستغرق بضع دقائق)
sudo docker pull bkimminich/juice-shop

# تشغيل Juice Shop على المنفذ 3000
sudo docker run -d -p 3000:3000 --name juice-shop bkimminich/juice-shop

الخطوة 3: فتحه في Firefox

  1. افتح Firefox في Kali VM
  2. انتقل إلى http://localhost:3000
  3. يجب أن ترى الصفحة الرئيسية لـ Juice Shop — متجر إلكتروني لبيع العصائر. أغلق لافتة الترحيب وإشعار الكوكيز.
✅

نقطة تحقق

إذا كنت ترى الصفحة الرئيسية لـ Juice Shop مع صور المنتجات (مثل Apple Juice و Banana Juice وغيرها)، فأنت جاهز للمتابعة!

المهمة 1: تحديد التقنيات المستخدمة

مهمتك الأولى هي معرفة ما هي التقنيات التي بُني بها هذا التطبيق.

1A: فحص رؤوس الاستجابة في Firefox

  1. في Firefox، تأكد أنك على صفحة http://localhost:3000
  2. اضغط F12 لفتح أدوات المطور. انقر على تبويب Network (الشبكة).
  3. اضغط Ctrl + R لإعادة تحميل الصفحة. انقر على أول طلب في القائمة.
  4. في لوحة التفاصيل، انقر على Headers (الرؤوس) وانزل إلى قسم Response Headers (رؤوس الاستجابة).
💡

رؤوس مهمة يجب البحث عنها

  • X-Content-Type-Options — يجب أن ترى nosniff
  • X-Frame-Options — يجب أن ترى SAMEORIGIN
  • Access-Control-Allow-Origin — يجب أن ترى * (يسمح لأي موقع بإرسال طلبات — مشكلة أمنية!)

1B: فحص مصدر الصفحة لمعرفة إطار العمل

  1. انقر بزر الماوس الأيمن على الصفحة واختر View Page Source (عرض مصدر الصفحة) أو اضغط Ctrl + U
  2. انزل إلى الأسفل وابحث عن وسوم <script>: runtime.js، polyfills.js، vendor.js، main.js
  3. ابحث أيضاً عن وسم <app-root> في قسم <body>
💡

تلميح

الملفات runtime.js و polyfills.js و vendor.js و main.js ووسم <app-root> هي العلامة المميزة لإطار عمل Angular (إطار عمل SPA من الفصل 3).

المهمة 2: استكشاف طلبات REST API في المتصفح

تتواصل تطبيقات الويب الحديثة من خلال REST API. لنشاهد ذلك يحدث مباشرة.

2A: فتح تبويب الشبكة

  1. اضغط F12 لفتح أدوات المطور، انقر على تبويب Network (الشبكة)
  2. انقر على زر فلتر XHR لعرض طلبات API فقط
  3. اضغط Ctrl + R لإعادة تحميل الصفحة

2B: مراقبة طلبات API

بدون النقر على أي شيء، فقط بتحميل الصفحة الرئيسية، يجب أن ترى عدة طلبات API. إليك بعض الطلبات الرئيسية:

طريقة HTTP مسار URL الوظيفة
GET /rest/admin/application-version الحصول على إصدار Juice Shop
GET /rest/products/search?q= البحث في جميع المنتجات
GET /rest/user/whoami التحقق من تسجيل دخول المستخدم

2C: النقر على منتج

انقر على أي بطاقة منتج في الصفحة الرئيسية. راقب تبويب Network — يجب أن ترى طلبات جديدة إلى /rest/products/{id}/reviews. لاحظ أن المراجعات تحتوي على عناوين بريد إلكتروني (مثل admin@juice-sh.op) — وهذا يعتبر كشف بيانات حساسة.

المهمة 3: قراءة بيانات JSON

تستخدم واجهات REST API صيغة JSON (JavaScript Object Notation) لتبادل البيانات بين العميل والخادم. JSON هي صيغة نصية خفيفة تنظم البيانات باستخدام أزواج المفتاح والقيمة. لنفحص بيانات JSON التي يرسلها Juice Shop.

3A: عرض JSON لإصدار التطبيق

  1. في تبويب Network (مع فلتر XHR)، ابحث عن طلب application-version وانقر عليه
  2. انقر على تبويب Response (الاستجابة) لرؤية بيانات JSON:

استجابة JSON المتوقعة:

{"version":"19.1.1"}

3B: عرض JSON لبحث المنتجات

  1. انقر على طلب /rest/products/search?q=، ثم انقر على Response
  2. سترى استجابة JSON تحتوي على جميع المنتجات. كل منتج يبدو هكذا:

مثال على كائن منتج:

{
  "id": 1,
  "name": "Apple Juice (1000ml)",
  "price": 1.99,
  "image": "apple_juice.jpg"
}

المهمة 3 — أدخل إجاباتك:

المهمة 4: فحص التخزين في المتصفح

يمكن للمتصفحات تخزين البيانات محلياً باستخدام الكوكيز (Cookies) و localStorage و sessionStorage. لنرى ما يخزنه Juice Shop في متصفحك.

4A: فتح مفتش التخزين

  1. اضغط F12 لفتح أدوات المطور
  2. انقر على تبويب Storage (التخزين) — إذا كان مخفياً، انقر على أيقونة « في نهاية شريط التبويبات
  3. وسّع Cookies ← http://localhost:3000 لرؤية جميع الكوكيز
  4. افحص أيضاً Local Storage و Session Storage
💡

ما يجب أن تراه

تحت Cookies، يجب أن ترى 3 إدخالات: language و welcomebanner_status و cookieconsent_status. تحقق من أعمدة HttpOnly و Secure.

استكشاف الأخطاء

❌ أمر Docker غير موجود

الحل: نفّذ sudo apt update && sudo apt install -y docker.io ثم sudo systemctl start docker.

❌ "Cannot connect to the Docker daemon"

الحل: نفّذ sudo systemctl start docker.

❌ صفحة Juice Shop لا تُحمّل

الحل: تحقق من تشغيل الحاوية: sudo docker ps. إذا لم تكن مدرجة، شغّلها: sudo docker start juice-shop.

❌ المنفذ 3000 قيد الاستخدام

الحل: نفّذ sudo docker stop juice-shop && sudo docker rm juice-shop، ثم أعد تنفيذ أمر docker run من الجزء 1.
