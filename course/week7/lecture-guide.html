Week 7 — Secure Development Methodologies I

CYB 236 · Ch. 17–20 · 75 min
Chapter 17 — Securing Modern Web Applications ~15 min
1The Castle Analogy— large attack surface, defend by priority, not uniformly
2Penetrate-and-Patch vs. Holistic Approach— reactive vs. proactive security culture
3The Three Pillars— Architecture → Code Review → Vulnerability Discovery (in order)
4Vulnerability Management & Risk Assessment— financial risk, ease of exploitation, data type, SLAs
5Regression Testing— ~25% of vulns are regressions; write a test after every fix
Chapter 18 — Secure Application Architecture ~20 min
1Analyzing Requirements for Hidden Risk— MegaMerch case: credentials, PII, auth model, financial data
2The NIST Finding— fixing at architecture phase costs 30–60× less than in production
3SSL vs. TLS & HTTPS— SSL is deprecated; TLS is the standard; all traffic must use HTTPS
4Password Hashing— plain text → MD5 → BCrypt / PBKDF2; slow algorithms resist brute-force
5Two-Factor Authentication (2FA)— attacker needs password AND physical device
Chapter 19 — Reviewing Code for Security ~20 min
1Archetypical vs. Logic Vulnerabilities— scanners find known patterns; logic bugs need human review
2How to Start — Git Review Flow— checkout master → pull → switch branch → diff
3The 5-Step Review Framework— client → API → dependencies → exposed APIs → remaining code
4Anti-Pattern 1: Blacklists— always incomplete; use whitelists instead
5Anti-Pattern 2: Boilerplate in Production— default configs are for dev, not security (MongoDB incident)
6Anti-Pattern 3: Trust-by-Default— one account for all modules; use least-privilege instead
7Anti-Pattern 4: Client/Server Coupling— separate concerns; communicate via typed JSON over HTTPS
Chapter 20 — Vulnerability Discovery ~20 min
1Why Discovery Is Still Needed— architecture + review still leave gaps in production
2Static Analysis— reads source without running it; fast but misses runtime & logic bugs
3Dynamic Analysis— runs the app; confirms real vulnerabilities; slower and costlier
4Regression Testing— Steve & Jed CSRF story; run on every push hook
5Responsible Disclosure— legal path for users to report; fix window before publishing
6Bug Bounty Programs— HackerOne / BugCrowd; cash rewards; pen testers + hunters
7The SSDL Wrap-Up— all techniques combined = Secure Software Development Life Cycle
