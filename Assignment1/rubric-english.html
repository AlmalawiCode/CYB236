Part C
Cross-Site Scripting (XSS) Attacks (Lab 3)
45 pts

💡 Expected Answers — DVWA XSS (Reference)

  • Q8 — Payload: <script>alert('XSS Works!')</script> (or any valid alert payload)
  • Q9 — Reflected XSS flow: Payload travels in URL query string → server reflects it in response HTML → browser executes it. Not stored anywhere permanently.
  • Q10 — Stored XSS reload: Payload was saved to the database on first submit. Every time the page loads, the server retrieves and outputs the script from DB, so the browser executes it again.
  • Q11 — Comparison: Reflected: URL only / victim must click link / one victim at a time. Stored: database / any visitor triggers it automatically / potentially all users. Stored is more dangerous.
  • Q12 — Bank scenario: All support agents who view that ticket are victims. Attacker receives their session cookies and can impersonate them (session hijacking), accessing customer data and admin functions.

📝 Grading Notes for Instructors

  • Accept any valid XSS payload in Q8 that correctly uses script tags, even if the alert message differs.
  • For Q3, Q6, Q9, Q10, Q12 — award partial credit generously if the student demonstrates understanding even with imprecise wording.
  • Screenshots need not be high-resolution; readability is the key criterion.
  • If a student's Kali VM shows a different Nmap version than expected in Q2, this is acceptable — award full marks as long as a valid version number is reported.
