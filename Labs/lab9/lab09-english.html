Lab 9: Defense in Practice — DVWA Security Levels

Topic 9 — Secure Development Methodologies III

DVWA Security Levels: Low → Medium → High

Lab Purpose

This lab shifts focus from attacking to defending. Students re-run the same attacks from Labs 3, 4, and 5 (XSS, CSRF, SQL Injection) but at increasing DVWA security levels. By observing how each defense stops or mitigates the attack, students understand concretely how defenses against XSS, CSRF, and XXE work — the focus of Topic 9.

Tools Required

DVWA Burp Suite (optional — for observing headers)

Planned Tasks

  • Task 1 — XSS Defense: Attempt the Stored XSS attack from Lab 3 at LOW, MEDIUM, and HIGH security levels. At each level, read the DVWA source code (View Source button) and identify what defense was added. Document: what changed in the code, and does the attack still work?
  • Task 2 — CSRF Defense: Attempt the CSRF attack from Lab 4 at LOW, MEDIUM, and HIGH security. At HIGH level, observe the CSRF token in the form. Explain what a CSRF token is and why it prevents the attack.
  • Task 3 — SQL Injection Defense: Attempt the SQL Injection attack from Lab 4 at LOW, MEDIUM, and HIGH. At HIGH level, observe parameterized queries in the source code. Explain the difference between string concatenation and parameterized queries.
  • Task 4 — Summary Table: Complete a comparison table: Attack | LOW (vulnerable?) | MEDIUM (defense applied?) | HIGH (fully protected?) | Defense technique used.

Learning Outcomes

  • Understand how input sanitization defends against XSS
  • Understand how CSRF tokens prevent cross-site request forgery
  • Understand how parameterized queries prevent SQL injection
  • Read and interpret security-relevant source code
Instructor Note: DVWA's "View Source" and "Compare" buttons are the key pedagogical tool here. Students should toggle between LOW and HIGH and read the code side by side. No new tools needed — this lab uses only DVWA.
